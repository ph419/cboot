## 问题分析

用户在"修改配置"菜单中编辑上下文窗口大小时，脚本抛出异常：
```
设置"contextWindow"时发生异常:"在此对象上找不到属性"contextWindow"。请确认该属性存在并且可设置。"
```

**根因**: `cboot.ps1:1632` 使用直接赋值 `$config.models[$i].contextWindow = $newContextOriginal` 设置 PSCustomObject 的属性。当该 model 对象在 JSON 中没有 `contextWindow` 字段（旧配置或未初始化该字段），PSCustomObject 不允许直接赋值不存在的属性。

**已有修复模式**: 代码中其他同类位置（行 607、685、698、744）均使用 `Add-Member -Force` 来安全添加/更新 PSCustomObject 属性。

## 实施计划

### Step 1: 修复 line 1632

将直接赋值改为 `Add-Member -Force` 模式：

```powershell
# 修改前:
$config.models[$i].contextWindow = $newContextOriginal

# 修改后:
$config.models[$i] | Add-Member -MemberType NoteProperty -Name "contextWindow" -Value $newContextOriginal -Force
```

### Step 2: 检查同类型问题

扫描所有对 PSCustomObject 的直接赋值，确认无类似遗漏。

### Step 3: 自测验证

- 模拟旧配置（model 对象无 contextWindow 字段），执行上下文窗口修改操作，确认不再报错
- 模拟新配置（model 对象已有 contextWindow 字段），确认正常更新
