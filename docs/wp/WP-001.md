## 目标

为 cboot 添加三项功能：
1. **配置初始化** — 配置文件不存在时交互式引导生成 `claude-config.json` 和对应的 settings 模板
2. **添加模型** — 主菜单中添加"添加模型"选项，交互式输入模型信息并持久化
3. **移除模型** — 主菜单中添加"移除模型"选项，方向键选择并确认删除

## 背景

当前 `claude-config.json` 完全依赖手工创建（通过 `install.ps1` 复制示例文件）。模型列表也需手动编辑 JSON 文件，而项目目录已有 `Add-ProjectDirectory` / `Remove-ProjectDirectory` 的交互式管理能力。需要将模型管理提升到同等水平。

## 实施计划

### Step 1: 新增 `Initialize-Config` 函数

**位置**: 替换当前第14-29行的"配置不存在则退出"逻辑

**行为**:
- 配置文件不存在时，进入交互式引导（而非直接退出）
- 步骤 1/3: 输入模型 ID 和显示名称（必填）
- 步骤 2/3: 选择默认权限（yes/no）
- 步骤 3/3: 输入初始项目目录（可选，可跳过）
- 生成 `claude-config.json`，包含一个模型、默认权限、可选项目目录
- 同时调用 `New-SettingsTemplate` 生成 settings 模板文件
- 完成后继续正常启动流程（不退出）

**交互流程**:
```
=== 欢迎使用 cboot ===
未找到配置文件，即将引导您完成初始配置。

步骤 1/3: 配置模型
请输入模型 ID（如 glm-5.1）: ___
请输入模型显示名称（如 GLM-5.1）: ___

步骤 2/3: 默认权限
是否默认允许所有操作？(Y/n): ___

步骤 3/3: 项目目录
请输入项目目录路径（留空跳过）: ___

✅ 配置文件已生成！
⚠️ 请编辑 settings-xxx.json 填入 API 密钥。
按任意键继续...
```

### Step 2: 新增 `New-SettingsTemplate` 辅助函数

**位置**: 在 `Save-Config` 函数之后（~第51行）

**职责**: 根据模型 ID 生成 settings 模板文件

**模板结构**（基于 `config/settings/settings-glm-5.1.example.json`）:
```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "YOUR_API_KEY_HERE",
    "ANTHROPIC_BASE_URL": "https://open.bigmodel.cn/api/anthropic",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "<model-id>",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "<model-id>",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "<model-id>"
  },
  "permissions": { "allow": [...], "deny": [...] },
  "language": "中文",
  "model": "<model-id>"
}
```

**参数**: `$modelId` — 模型标识符，用于替换模板中的占位符

### Step 3: 新增 `Add-Model` 函数

**位置**: 在 `Add-ProjectDirectory` 之前（~第475行）

**复用 `Add-ProjectDirectory` 的交互模式**（第476-532行）

**输入字段**:
- `id`: 模型标识符（必填，如 "glm-5.1"）
- `name`: 显示名称（必填，如 "GLM-5.1"）
- `configFile`: settings 文件名（必填，如 "settings-glm-5.1.json"）

**验证**:
- id 不能为空，不能重复
- name 不能为空
- configFile 不能为空
- 如果 configFile 对应的文件不存在，自动调用 `New-SettingsTemplate` 生成模板

**交互流程**:
```
=== 添加模型 ===

当前配置的模型:
  - glm-5.1 (GLM-5.1)

输入模型 ID（如 glm-5.1）: ___
输入模型显示名称（如 GLM-5.1）: ___
输入配置文件名（如 settings-glm-5.1.json）: ___

✅ 模型添加成功！
⚠️ 已生成 settings-xxx.json 模板，请填入 API 密钥。
按任意键继续...
```

### Step 4: 新增 `Remove-Model` 函数

**位置**: 在 `Add-Model` 之后

**复用 `Remove-ProjectDirectory` 的交互模式**（第534-629行）

**特殊逻辑**:
- 如果只剩一个模型，禁止删除并提示
- 删除后如果 `defaultModel` 指向被删除的模型，自动切换到第一个剩余模型

**交互流程**:
```
=== 移除模型 ===

选择要移除的模型:
  1. glm-5.1 (GLM-5.1)
  2. glm5 (GLM-5)

使用方向键导航，回车选择，ESC取消

=== 确认移除 ===
你确定要移除以下模型吗:
  glm-5.1 (GLM-5.1)
此操作无法撤销。
  是，移除它
  否，取消
```
