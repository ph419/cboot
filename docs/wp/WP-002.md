## 背景

WP-001（配置初始化与模型管理功能）已标记为 ✅ 完成，新增了 `Initialize-Config`、`Add-Model`、`Remove-Model` 三个函数及主菜单扩展（+415行）。代码审查发现 WP-001 的实现存在 1 个关键 Bug 和 3 个潜在问题。

## Bug 分析

### Bug #1 [CRITICAL] - Initialize-Config 为死代码

**位置**: `cboot.ps1:15-29` 与 `cboot.ps1:251-254`

**问题**: 存在两段"配置文件不存在"的检查逻辑：
- 第 15-29 行：检查配置不存在 → 显示错误 → **exit 退出**
- 第 251-254 行：检查配置不存在 → 调用 `Initialize-Config`

第 15-29 行的 `exit` 会在 `Initialize-Config` 被调用之前就终止脚本。因此 `Initialize-Config` 函数（第 136-249 行）永远不会被执行 — 是死代码。

**根因**: WP-001 计划中明确写了"替换当前第14-29行的'配置不存在则退出'逻辑"，但实际实现中**没有替换**，而是保留了原始的退出逻辑，又在后面新增了初始化调用。

### Bug #2 [MEDIUM] - Remove-Model 中删除确认使用过期的 selectedIndex

**位置**: `cboot.ps1:849-856`

**问题**: 确认删除时重新 `Get-Config` 读取配置，但使用的是上一次循环中的 `$selectedIndex`。如果外部修改了配置导致模型顺序变化，会删除错误的模型。

### Bug #3 [MEDIUM] - Remove-ProjectDirectory 同样的过期索引问题

**位置**: `cboot.ps1:1011-1017`

**问题**: 与 Bug #2 相同的模式，确认删除时用 `$selectedIndex` 而非路径匹配。

### Bug #4 [LOW] - 排序刷新后导航位置可能跳转

**位置**: `cboot.ps1:1168`, `cboot.ps1:1202`

**问题**: 选择模型/项目后增加使用计数并刷新排序列表，`$modelIndex`/`$dirIndex` 不变但列表顺序已变，导致 ESC 回退时高亮位置可能跳到不同项目。

## 实施计划

### Step 1: 修复 Bug #1 - 删除死代码检查 [CRITICAL]

**修改文件**: `cboot.ps1`

**删除第 15-29 行**（旧版配置检查逻辑）:
```powershell
# 这段代码导致 Initialize-Config 成为死代码，必须删除
if (-not (Test-Path $configPath)) {
    Clear-Host
    Write-Host "=== 配置错误 ===" -ForegroundColor Red
    ...
    exit  ← 总是在 Initialize-Config 之前退出
}
```

**验证**: 删除配置文件后运行脚本，确认进入交互式引导流程而非退出。

### Step 2: 修复 Bug #2 - Remove-Model 用 ID 匹配 [MEDIUM]

**修改文件**: `cboot.ps1:849-856`

**将索引匹配改为 ID 匹配**:
```powershell
# 原代码（索引匹配 - 危险）:
$newModels = @()
for ($i = 0; $i -lt $config.models.Count; $i++) {
    if ($i -ne $selectedIndex) {
        $newModels += $config.models[$i]
    }
}

# 修复后（ID 匹配 - 安全）:
$newModels = @($config.models | Where-Object { $_.id -ne $removedModelId })
```

### Step 3: 修复 Bug #3 - Remove-ProjectDirectory 用路径匹配 [MEDIUM]

**修改文件**: `cboot.ps1:1011-1017`

**将索引匹配改为路径匹配**:
```powershell
# 原代码（索引匹配 - 危险）:
$newProjects = @()
for ($i = 0; $i -lt $config.projects.Count; $i++) {
    if ($i -ne $selectedIndex) {
        $newProjects += $config.projects[$i]
    }
}

# 修复后（路径匹配 - 安全）:
$newProjects = @()
foreach ($p in $config.projects) {
    $path = if ($p -is [string]) { $p } else { $p.path }
    if ($path -ne $removedPath) {
        $newProjects += $p
    }
}
```

### Step 4: 修复 Bug #4 - ESC 回退时重定位高亮 [LOW]

**修改文件**: `cboot.ps1`

在 dirLoop 的 ESC 回退处（~第 1184-1191 行），根据当前选中模型的 ID 重新查找在新排序列表中的索引：
```powershell
# 在 break dirLoop 之前添加:
for ($i = 0; $i -lt $sortedModels.Count; $i++) {
    if ($sortedModels[$i].id -eq $currentModelId) {
        $modelIndex = $i
        break
    }
}
```
