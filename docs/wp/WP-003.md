## 问题分析

当前 cboot.ps1 的「添加模型」功能（`Add-Model` 函数，第 663-755 行）存在两个核心问题：

### 问题 1：无法取消退出

`Add-Model` 使用 `Read-Host` 逐步输入，每个字段都要求非空。用户一旦进入该流程就无法取消，只能填完所有信息或强制关闭窗口。`Initialize-Config`（第 118-232 行）同样存在此问题。

### 问题 2：参数不完整

只输入 id/name/configFile 三个基础字段，生成的 settings 模板中：
- `ANTHROPIC_AUTH_TOKEN` 固定为 `YOUR_API_KEY_HERE`
- `ANTHROPIC_BASE_URL` 固定为 `https://open.bigmodel.cn/api/anthropic`
- `ANTHROPIC_DEFAULT_OPUS_MODEL` = `ANTHROPIC_DEFAULT_SONNET_MODEL` = `ANTHROPIC_DEFAULT_HAIKU_MODEL` = modelId

用户每次都要手动编辑 settings 文件才能正常使用。

### 问题 3：模板硬编码

`New-SettingsTemplate`（第 36-116 行）用 hashtable 硬编码整个 settings 结构，无法继承用户已有的自定义配置（如额外的 permissions、plugins 等）。应以已有 settings 文件（如 `settings-glm-5.1.json`）为模板。

## 实施计划

### Step 1: 改造 `New-SettingsTemplate` 为模板复制模式

**文件**: `cboot.ps1` 第 36-116 行

**改动**:

1. 查找 `~/.claude/settings-*.json` 中第一个存在的文件作为模板
2. 读取模板 JSON → 解析 → 替换关键 env 字段 → 写入新文件
3. 替换的字段由调用方传入
4. 其他字段（hooks, permissions, enabledPlugins 等）原样保留
5. 无模板时 fallback 到现有硬编码模板（保留兜底）

新函数签名：
```powershell
function New-SettingsTemplate {
    param(
        [string]$modelId,
        [string]$configFile,
        [string]$authToken,
        [string]$baseUrl,
        [string]$opusModel,
        [string]$sonnetModel,
        [string]$haikuModel
    )
    # 1. 找模板：遍历 ~/.claude/settings-*.json，排除目标文件自身
    # 2. 无模板时 fallback 到现有硬编码模板
    # 3. 读取模板 JSON → ConvertFrom-Json
    # 4. 替换 env 中的指定字段
    # 5. 设置 model = $modelId
    # 6. ConvertTo-Json -Depth 10 → 写入
}
```

### Step 2: 改造 `Add-Model` 添加 ESC 退出和参数输入

**文件**: `cboot.ps1` 第 663-755 行

**改动**:

1. **ESC 退出机制**：将每个 `Read-Host` 调用替换为自定义函数，支持输入空行时提示确认取消，确认后返回主菜单。

2. **新增参数输入**（在 configFile 输入之后、生成模板之前）：
   - `ANTHROPIC_AUTH_TOKEN`（必填，显示掩码提示）
   - `ANTHROPIC_BASE_URL`（有默认值 `https://open.bigmodel.cn/api/anthropic`，空行取默认值）
   - `ANTHROPIC_DEFAULT_OPUS_MODEL`（默认 = modelId，空行取默认值）
   - `ANTHROPIC_DEFAULT_SONNET_MODEL`（默认 = modelId，空行取默认值）
   - `ANTHROPIC_DEFAULT_HAIKU_MODEL`（默认 = modelId，空行取默认值）

3. **调用改造后的 `New-SettingsTemplate`**：传入所有收集到的参数。

### Step 3: 改造 `Initialize-Config` 保持一致

**文件**: `cboot.ps1` 第 118-232 行

**改动**：
1. 添加与 Add-Model 相同的 5 个参数输入步骤
2. 调用改造后的 `New-SettingsTemplate` 并传入参数
3. 同样支持空行取消
