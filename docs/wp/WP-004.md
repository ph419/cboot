## 问题分析

### 问题 1：缺少上下文窗口设置

cboot 用户使用第三方 API（如智谱）时，模型上下文窗口可能受限（如 200K）。Claude Code 默认使用 1M 上下文窗口，需通过以下环境变量控制：

- `CLAUDE_CODE_DISABLE_1M_CONTEXT = "1"` — 禁用 1M 上下文
- `CLAUDE_CODE_AUTO_COMPACT_WINDOW = "<tokens>"` — 自动压缩窗口大小（建议设为上下文的 90%）

目前 cboot 初始化和添加模型流程中没有上下文窗口设置，用户必须手动编辑 JSON 文件。

### 问题 2：缺少配置编辑入口

已初始化的用户无法通过交互式菜单修改 settings-*.json 中的 env 变量（API 密钥、Base URL、模型映射等）。每次修改都需要手动编辑 JSON 文件，容易出错。

### WP-004-1-impl: 上下文窗口功能

#### Step 1: 新增 Parse-ContextWindow 函数

在 `cboot.ps1` 函数定义区（~165 行 `New-SettingsTemplate` 之前）添加：

```powershell
function Parse-ContextWindow {
    param([string]$Input)

    if ([string]::IsNullOrWhiteSpace($Input)) {
        return @{ size = 1000000; original = "1M" }
    }

    $Input = $Input.Trim().ToUpper()

    # 匹配 "200K" 格式
    if ($Input -match '^(\d+(?:\.\d+)?)\s*K$') {
        $value = [double]$Matches[1]
        return @{ size = [math]::Floor($value * 1000); original = "$($value)K" }
    }

    # 匹配 "1M" 格式
    if ($Input -match '^(\d+(?:\.\d+)?)\s*M$') {
        $value = [double]$Matches[1]
        return @{ size = [math]::Floor($value * 1000000); original = "$($value)M" }
    }

    return $null  # 无效输入
}
```

行为说明：
- `200K` → size=200000, original="200K"
- `1M` → size=1000000, original="1M"
- 空输入 → size=1000000, original="1M"（默认 1M）
- 无效输入 → 返回 $null


```
  2. Base URL    : https://open.bigmodel.cn/api/anthropic
  3. Opus 模型   : glm-5.1
  4. Sonnet 模型  : glm-5.1
```
