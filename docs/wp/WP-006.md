## 目标

移除 cboot.ps1 中 3 处将上下文窗口值乘以 0.9 的逻辑，改为直接使用用户填写的原始值。

## 问题分析

当前 cboot 在设置 `CLAUDE_CODE_AUTO_COMPACT_WINDOW` 时，会将用户填写的上下文窗口值乘以 0.9。例如用户填 200K，实际设为 180000。用户决定去掉这个比例，填多少就设多少。

## 实施计划

### Step 1: 修改 New-SettingsTemplate 函数（行 298）

**文件**: `cboot.ps1` 行 298

```powershell
# 当前:
$compactWindow = [math]::Floor($contextWindowSize * 0.9)
# 改为:
$compactWindow = $contextWindowSize
```

### Step 2: 修改更新模型配置逻辑（行 1264）

**文件**: `cboot.ps1` 行 1264

```powershell
# 当前:
$compactWindow = [math]::Floor($contextWindowSize * 0.9)
# 改为:
$compactWindow = $contextWindowSize
```

### Step 3: 修改编辑上下文窗口逻辑（行 1616）

**文件**: `cboot.ps1` 行 1616

```powershell
# 当前:
$compactWindow = [math]::Floor($newContextSize * 0.9)
# 改为:
$compactWindow = $newContextSize
```
