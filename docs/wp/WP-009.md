## 问题分析

Claude Code v2.1.160 支持通过 settings JSON 顶级字段 `teammateDefaultModel` 配置 teammate 默认模型。当前 cboot 完全不支持此字段，用户无法通过启动器配置 teammate 模型。

**技术要点**：
- `teammateDefaultModel` 是顶级字段，不是 `env` 子字段
- 类型为 `string | null`，null 表示使用系统默认（Sonnet）
- 旧版 settings.json 可能不存在此属性，需用 `Add-Member -Force` 兼容

## 实施计划

### Step 1: 更新 `New-SettingsTemplate` 硬编码模板

**文件**: `cboot.ps1` L224-282

在硬编码模板的 `statusLine` 之前添加：
```powershell
teammateDefaultModel = $null
teammateMode = "auto"
```

### Step 2: 增加 `New-SettingsTemplate` 参数

**文件**: `cboot.ps1` L189-198

在函数参数列表中增加：
```powershell
[string]$teammateModel = $null
```

### Step 3: 添加 teammate 配置注入

**文件**: `cboot.ps1` L301 之后（上下文窗口设置之后）

```powershell
# teammate 配置
$template | Add-Member -MemberType NoteProperty -Name "teammateDefaultModel" -Value $teammateModel -Force
$template | Add-Member -MemberType NoteProperty -Name "teammateMode" -Value "auto" -Force
```

### Step 4: 更新 `Add-Model` 流程

**文件**: `cboot.ps1` L1219 之后（HAIKU_MODEL 输入之后）

增加 teammate 模型输入：
```powershell
# 输入 TEAMMATE_MODEL（默认 = null，即使用系统默认）
$teammateModel = Read-HostWithCancel "输入 Teammate 默认模型（留空使用系统默认）"
if ([string]::IsNullOrWhiteSpace($teammateModel)) {
    $teammateModel = $null
}
```

同时更新 `New-SettingsTemplate` 调用处（L1253），传递新参数。

### Step 5: 更新 `Initialize-Config` 流程

**文件**: `cboot.ps1` L380 之后（HAIKU_MODEL 输入之后）

与 Add-Model 一致，增加 teammate 模型输入。同时更新确认屏幕和 `New-SettingsTemplate` 调用。

### Step 6: 更新 `Edit-ModelConfig` 字段列表

**文件**: `cboot.ps1` L1509-1517

在 `$fieldItems` 数组中，在 "Haiku 模型" 之后、"上下文窗口" 之前插入：
```powershell
"Teammate 模型: $(if ($settings.teammateDefaultModel) { $settings.teammateDefaultModel } else { '系统默认' })"
```

更新 `$fieldSelectedIndex` 的 maxIndex（从 6 改为 7），并在 switch 中增加 case 5（Teammate 模型编辑），将原 case 5（上下文窗口）改为 case 6。

编辑逻辑使用 `Add-Member -Force` 写入，兼容旧文件。

### Step 7: 更新示例 settings 文件

4 个 `config/settings/settings-*.example.json` 文件，在 `"model"` 字段之前添加：
```json
"teammateDefaultModel": null,
"teammateMode": "auto",
```
